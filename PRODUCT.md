# Product

## Register

product

## Users

Real-estate investors, analysts, and engineering reviewers evaluating the project as a portfolio piece. They arrive mid-workflow: load a deal, run the committee, read the verdict, drill into the workstreams that drove it. Secondary audience is hiring managers skimming screenshots — the first screen has to communicate "credible financial tool" in five seconds.

## Product Purpose

A deterministic institutional diligence engine with a dashboard surface: run a 77-workstream investment committee on a rental property, inspect scores, scenarios, risks, and the memo, and review past runs. The UI's job is to make dense quantitative output legible and trustworthy — never to decorate it.

## Brand Personality

Calm, numerate, exact. Stripe-dashboard energy: light surfaces, generous whitespace, one confident accent, numbers treated as first-class typography (tabular figures, right-aligned columns). Confidence through restraint, not density or darkness.

## Anti-references

- Crypto/trading dashboards: no neon green/red, no glow, no dark-mode-for-coolness.
- Generic SaaS admin templates: no identical stat-card rows, no bootstrap-admin blue-on-white, no rounded-everything.
- AI-generated tells: no gradient text, glassmorphism, purple-blue gradients, identical icon-card grids, or side-stripe accent borders.

## Design Principles

1. **The number is the interface.** Scores, ratios, and dollar figures carry the page; typography and alignment do the explaining before any chrome does.
2. **Verdicts read at a glance.** Recommendation states (BUY / NEGOTIATE / WATCHLIST / PASS) have one consistent, quiet visual encoding everywhere they appear.
3. **Calm earns trust.** One accent, warm-neutral grays, soft depth; nothing flashes, nothing glows.
4. **Density with air.** Tables are compact but never cramped; whitespace separates concerns instead of borders wherever possible.
5. **Every state is designed.** Loading, empty, and error states get the same care as the happy path.

## Accessibility & Inclusion

WCAG 2.1 AA: body text ≥4.5:1, large text ≥3:1, visible focus states, full keyboard navigation, `prefers-reduced-motion` honored. Recommendation states never rely on color alone (always paired with the label text).
